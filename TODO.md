这里记载长期规划，无需频繁检查与更新：

## 核心目标
创造轻量、Live 性能良好的终端 UI 库。
直接面向动态渲染循环，不引入复杂的底层滚动逻辑。
语法对齐 Rich，保持 AI 友好性

## 优先级 1：监听循环 (Listening Loop) - [底层逻辑]
[x] **异步输入系统**：实现基于 `threading` 或 `selectors` 的非阻塞键盘输入监听。
[x] **按键映射 (Key Mapping)**：支持方向键、组合键及特殊功能键的标准化解析。
[x] **输入法支持研究**：针对终端下中文/复杂字符输入的处理。
[ ] **事件分发**：建立简单的事件回调机制（如 `on_key`, `on_click`）。

## 优先级 2：组件抽象 (Component Abstraction) - [基础套件]
[x] **基础容器**：实现 `Panel` (边框容器) 和 `Group` (组合容器)。
[ ] **UI 原子**：实现 `StaticText` (静态文本)、`LiveText` (动态绑定文本)。
[x] **输入组件**：实现简易的 `InputBox` 和 `Button`。
[ ] **资产占位符**：优化图像/字体预加载的封装。

## 优先级 3：动态布局管理 (Layout & Logic) - [对齐 Rich]
[x] **简化布局语法**：学习 HTML 模式，但在终端空间进行简化。
[ ] **动态内容绑定**：实现页面内容的动态更新逻辑，而非仅渲染静态内容。
[ ] **区域管理**：优化 `RenderEngine` 对屏幕区域的划分与管理。

## 优先级 4：样式重构 (Style Refactoring) - [代码整理]
[x] **易用性改进**：重构渲染函数的参数（如颜色、样式、对齐），参考 Textual 的简洁性。
[x] **样式对象化**：引入 `Style` 类，支持样式继承和复用。
[ ] **代码去重**：统一 `push_xxx` 系列方法的底层调用接口。
